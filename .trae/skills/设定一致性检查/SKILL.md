# 设定一致性检查

## 目标

对指定章节或正文片段做一致性稽核，找出“吃书/前后矛盾/命名不统一/能力边界破裂”，并给出可直接落地的修正方案。

## 何时调用

- 用户说“检查设定矛盾/有没有吃书/帮我校对设定”
- 用户写完一章要发布前做质检
- 用户要回改前文，担心影响后续伏笔与设定

## 输入

- 章节文件路径或正文片段
- 若未提供路径：先从 `正文/` 里定位最新章节

## 必读上下文（尽量从仓库读取）

1. `设定/`（修炼体系、地理组织、规则边界）
2. `人物/`（涉及角色的角色卡）
3. `笔记/`（卷纲、章纲、时间线类记录）

## 输出格式

按严重程度输出问题清单，每条包含：

1. 问题类型（矛盾/命名/边界/动机/时间线）
2. 证据（引用正文与设定的关键句，说明冲突点）
3. 修正策略（优先“最小改动”）
4. 建议改动（给出可直接替换的句子或段落草案）

## 行动准则

- 若冲突发生：以 `设定/` 与 `人物/` 为准，正文做最小改动修复
- 若设定缺失导致歧义：优先补充到对应 `设定/` 或 `人物/` 文件（用 `apply_patch` 更新），同时给出正文侧的过渡写法
- 不做大重写，除非用户明确要求“整章重写”
