# 创建章纲

## 目标

基于仓库内`大纲.md`与设定资料，为指定范围的章节生成“可直接写正文”的章纲，并把章纲写入仓库中对应的文件夹。
**风格核心**：番茄爆款风，迪化流，无敌流。

## 何时调用

- 用户要求“出章纲/拆章/把前 N 章拆成章纲/给第 X 章章纲”
- 用户强调“章纲需要落盘/放到文件夹里”

## 必读上下文

1. `大纲.md`（必须严格遵循其中的“章前施工单”逻辑）
2. `笔记/` 下的伏笔与时间线（如果存在）

## 落盘规则

### 目录规则
- 基础目录：`章纲/`
- 卷目录：`章纲/<卷名>/`
  - 卷名请参考大纲：
    - `第一卷_归墟有个正常人`
    - `第二卷_师尊下山，全员迪化`
    - `第三卷_解释权之争`
    - `第四卷_我只是扫了个地`

### 文件命名
- `{NNN}.<章名>.md`（如 `001.洗脚水灭世.md`）

## 章纲模板（Markdown 表格版）

每章必须包含以下内容，直接写入文件：

```markdown
# {章节序号} {章节名}

## 1. 章前施工单（核心）

| 检查项 | 内容 |
| :--- | :--- |
| **本章生活目标** | {庸俗日常目标，如：给白菜浇水} |
| **外界阻碍（反派）** | {宏大恐怖危机，如：灭世魔尊降临} |
| **主角动作（金手指）** | {物理生活动作，如：泼洗脚水} |
| **脑补反馈（爽点）** | {弟子/反派的震惊，如：混沌神水湮灭魔尊} |
| **资源变化** | {物资-1，威望+1000} |
| **章末钩子** | {下章悬念，如：有人来查水表} |

## 2. 剧情细纲（镜头语言）

1.  **镜头一（铺垫）：** 主角出场，展示生活烦恼（字数占比 20%）。
    *   *细节：* {具体描写方向}
2.  **镜头二（冲突）：** 危机逼近，反派/外界视角渲染恐怖气氛（字数占比 30%）。
    *   *细节：* {具体描写方向}
3.  **镜头三（解决）：** 主角无意间出手，动作降维打击（字数占比 30%）。
    *   *细节：* {具体描写方向}
4.  **镜头四（脑补）：** 配角自行攻略，完成逻辑闭环（字数占比 10%）。
    *   *细节：* {具体描写方向}
5.  **镜头五（收束）：** 资源结算，抛出钩子（字数占比 10%）。
    *   *细节：* {具体描写方向}

## 3. 信息与伏笔
*   **新增信息**：{本章引入的新设定/人物}
*   **伏笔埋藏**：{本章留下的坑}
*   **伏笔回收**：{本章填上的坑}
```

## 多章生成策略

如果是生成多章（如“前10章”）：
1.  先规划每章的“生活目标”和“外界阻碍”，确保不重复且有递进。
2.  **每10章**必须安排一个“小高潮”（实物奖励 + 势力洗牌）。
3.  逐章生成文件，并在对话中列出清单。
